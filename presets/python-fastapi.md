## Code Conventions

### Naming

| Item | Convention | Example |
|------|-----------|---------|
| Files | snake_case | user_service.py |
| Functions/Methods | snake_case | get_user_by_id |
| Classes | PascalCase | UserService |
| Constants | SCREAMING_SNAKE_CASE | MAX_RETRIES |
| Variables | snake_case | current_user |
| Route paths | kebab-case | /api/user-profiles |

### Code Style

- **Formatter:** ruff format (or black)
- **Max Line Length:** 88 (ruff/black default)
- **Indentation:** 4 spaces
- **Import Order:** stdlib → third-party → local (enforced by ruff/isort)

### Patterns & Conventions

- **Error Handling:** Custom exception classes, FastAPI exception handlers, HTTPException for API errors
- **Logging:** structlog or standard logging with structured output
- **Configuration:** pydantic-settings for environment config, validated at startup
- **API Design:** RESTful, JSON request/response, Pydantic models for validation
- **Dependency Injection:** FastAPI Depends() for shared resources (DB sessions, auth, etc.)

## Testing

### Strategy

- **Unit Tests:** In tests/unit/, testing services and schemas
- **Integration Tests:** In tests/integration/, testing full HTTP with TestClient
- **Test Framework:** pytest
- **Mocking:** unittest.mock, pytest-mock
- **Coverage Target:** 80% (measured with pytest-cov)
- **Fixtures:** Shared fixtures in conftest.py

### Test Naming Convention

`test_should_return_user_when_id_exists`, `test_raises_404_when_user_not_found`

## Quality Gates

## Constraints & Non-Negotiables

- All request/response bodies must use Pydantic models
- All endpoints must have type hints
- No bare `except:` — always catch specific exceptions
- Environment variables must be validated with pydantic-settings
- Database access only through the service layer (not in routes directly)
