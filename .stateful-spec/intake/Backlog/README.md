# intake / Backlog — inbound ideas

Raw feature/improvement ideas not yet shaped into a numbered opportunity. One file per idea,
`kebab-case.md`, with the READY frontmatter (see [`../README.md`](../README.md)). At triage, a
`ready` item is promoted into [`../../backlog.md`](../../backlog.md) as a stable `O-NNN` entry.

## Example item

```markdown
---
status: draft
title: Export the current configuration as a shareable file
origin: idea
---

Let the user export the current settings/config as a shareable snippet, so a preset can be shared
or checked into a repo. Mirrors the import path the project already has. Shape the scope (which
fields, round-trip vs export-only) before promoting to O-NNN.
```

> `draft` until shaped; flip to `ready` when it is concrete enough to triage. Do **not** assign an
> `O-NNN` here — numbering belongs to [`../../backlog.md`](../../backlog.md).
